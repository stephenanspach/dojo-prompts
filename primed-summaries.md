# Primed Summaries

Take one or more existing SRTs (or Scribe JSON files) in any source language and
produce, for each, a **summary SRT** — a smaller SRT where each entry covers many
original sentences and the text is a 1–3 sentence **English summary** of what's
said in that span. This is not a translation: the AI groups sentences into
topical chunks and writes fresh English summaries.

The output drives downstream "primed listening" audio (English preview → original
audio for the chunk → next preview → ...).

## Usage

The user provides one or more paths to Scribe JSON files or SRT files. Optionally
the source language can be specified inline (e.g. `/primed-summaries file.json Japanese`).

**Always prefer JSON over SRT** — if a Scribe JSON file exists alongside the SRT
(same basename), use the JSON. Only use an SRT directly if no JSON is available.

## Architecture

**One subagent per episode.** Each input file is handed to a single subagent that
runs the *entire* pipeline (`prepare → premise → window loop → finalize`) inline,
in its own isolated work dir. The orchestrator (you, in the main context) never
reads transcript text — it only resolves inputs, computes paths via the helper,
and collects each subagent's short result.

This is **consistent regardless of episode count**: the per-episode pipeline is
identical whether there is 1 input or 20. The only thing that varies is
concurrency — if the agent can run independent sub-tasks at once, spawn the
episode subagents in parallel; otherwise run them sequentially. Either way the
result is the same.

The window loop runs **inline inside each episode subagent** — it does *not* spawn
further subagents (one level of subagent nesting only). The Python helper
(`dojo-prompts/scripts/srt_summarize.py`) handles SRT/JSON parsing, path naming,
work-dir cleanup, the sliding-window state, JSON validation/repair, the
core-region filter, the fallback path, and final SRT assembly.

`WINDOW_SIZE=40` sentences per window, `CORE_SIZE=25` core region — chunks ending
past sentence 25 of a non-final window are discarded so chunk boundaries fall on
natural topic transitions, not on window edges.

### Isolation & safety

- Each episode gets its own work dir under `/tmp/primed-summaries/`, generated by
  the helper's `make-workdir` command (sanitized stem + a hash of the absolute
  input path + a per-run nonce). Two same-named files in different folders, and
  concurrent runs of the same file, never collide.
- The helper owns its work dir via a `.primed-summaries-workdir` marker. **Never
  run `rm -rf` yourself** — `prepare` clears stale state safely, and refuses to
  touch any directory it doesn't own.
- **Quote every dynamic shell argument** with `shlex.quote` semantics — the input
  path, output path, work dir, and the source-language string can all contain
  spaces, apostrophes, or shell metacharacters (e.g. `Bob's episode.srt`). Naive
  single-quoting is **not** enough: a literal `'Bob's episode.srt'` breaks the
  command. The `'<PLACEHOLDER>'` notation in the command examples below is
  shorthand for "the shell-quoted value" — when you substitute a real path,
  escape it with `shlex.quote` (or pass argv as a list so no shell quoting is
  needed at all), do not just drop it inside the surrounding single quotes. Only
  fixed literals (mode names, the script path) are exempt.

## Workflow — orchestrator (main context)

Do **not** read any transcript, SRT body, or full JSON into the main context. Your
job is only to set up each episode and spawn its subagent.

### 1. Gather and resolve inputs

For each path the user gave:
- If it's an SRT and a Scribe JSON with the same basename exists alongside it,
  use the JSON instead. Otherwise use the path as given.
- Reject the batch if two inputs resolve to the **same absolute path**.

### 2. Compute paths for every episode (helper-owned naming)

For each resolved input, run:

```bash
python3 dojo-prompts/scripts/srt_summarize.py make-workdir '<input_path>'
```

It prints three lines; read each as `KEY=VALUE`, taking **everything after the
first `=`** as the raw literal value (values may contain spaces or `=`):

```
WORKDIR=/tmp/primed-summaries/<episode-id>
OUTPUT=<input-dir>/<lang-stripped-stem>.summary.en.srt
OUTPUT_ALT=<input-dir>/<full-stem>.summary.en.srt
```

Never construct these names yourself — only choose between `OUTPUT` and
`OUTPUT_ALT` in the next step.

### 3. Output-path collision preflight

Default each episode to its `OUTPUT`. If two inputs share the same `OUTPUT`
(e.g. `ep.ja.srt` and `ep.ko.srt` both → `ep.summary.en.srt`), switch *those*
episodes to their `OUTPUT_ALT` (which keeps the language code). If a collision
still remains after that, **reject the batch** with a clear message rather than
risk parallel clobbering.

### 4. Spawn one episode subagent per input

Spawn a subagent (Task tool, `subagent_type: "general-purpose"`) for each episode.
The subagent inherits the current session's model — do not specify one. If the
agent supports concurrent sub-tasks, spawn them in parallel; otherwise run them
one at a time. Give each subagent the **episode subagent prompt** below, filled
in with that episode's `WORKDIR`, input path, chosen output path, and the source
language (if the user supplied one).

### 5. Collect results

Each subagent returns only:

```json
{ "output_path": "...", "failed_windows": false, "warnings": [] }
```

Surface to the user: each output path, any `warnings` (e.g. a long-input warning),
and any episodes where `failed_windows` is true (those spans got placeholder
summaries and may need a manual pass). **Never** echo transcript text.

## Episode subagent prompt

Give the subagent the four values (work dir, input path, output path, language)
and these instructions. Every command must shell-quote its path and language
arguments.

```
You process ONE episode for the primed-summaries skill, end to end, in your own
isolated work dir. You read transcript text; the orchestrator does not — so keep
your final reply tiny (see step 6). Run the helper with --work-dir set to your
work dir on EVERY call. The '<...>' tokens below are placeholders: substitute the
real value and shell-quote it with shlex.quote (an input like Bob's episode.srt
will break naive single quotes) — or pass argv as a list so no quoting is needed.

Inputs you were given:
- WORKDIR:  <work dir>
- INPUT:    <input path>
- OUTPUT:   <output path chosen by the orchestrator — use it verbatim>
- LANGUAGE: <source language, or "unknown">

1. Prepare (also clears/creates the work dir safely — do NOT run rm -rf):
     python3 dojo-prompts/scripts/srt_summarize.py --work-dir '<WORKDIR>' prepare '<INPUT>'
   If the output contains a line starting with "WARNING:", remember it for step 6.

2. Determine the source language. If LANGUAGE is "unknown", read the first ~30
   lines of '<WORKDIR>/full_transcript.txt' and identify it. Then read the whole
   '<WORKDIR>/full_transcript.txt' and write a 2-4 sentence English PREMISE to
   '<WORKDIR>/premise.txt' covering:
     - the genre/format (lecture, interview, vlog, anime episode, podcast, etc.)
     - who's speaking (count, named hosts/characters if identifiable, roles)
     - the overall topic/setting/arc
     - the tone (casual, formal, comedic, instructional, etc.)
   Write ONLY the premise (no headers, no commentary). Names and specifics matter.

3. Window loop — repeat until next-window prints DONE:
     a. python3 dojo-prompts/scripts/srt_summarize.py --work-dir '<WORKDIR>' next-window '<LANGUAGE>'
        Output is one of:
          DONE                                         -> go to step 4
          PARTIAL cursor=N window_size=M ... brief=...  -> continue
          FINAL   cursor=N window_size=M ... brief=...  -> continue
     b. Read '<WORKDIR>/window_brief.txt' and follow its instructions: group the
        window's sentences into contiguous topical chunks (3-15 sentences each)
        and write English summaries.
     c. Write ONLY the JSON array (no markdown fences, no commentary) to
        '<WORKDIR>/window_output.json'.
     d. python3 dojo-prompts/scripts/srt_summarize.py --work-dir '<WORKDIR>' accept '<WORKDIR>/window_output.json'
        If accept exits non-zero, retry b–d (re-read the brief, rewrite the JSON)
        up to 3 times. If it still fails, run the fallback and continue:
          python3 dojo-prompts/scripts/srt_summarize.py --work-dir '<WORKDIR>' fallback

4. Finalize to the EXACT output path you were given (do not derive or change it):
     python3 dojo-prompts/scripts/srt_summarize.py --work-dir '<WORKDIR>' finalize '<OUTPUT>'

5. Spot check: read a few entries (start, middle, end) of '<OUTPUT>' to confirm
   timestamps span sensible ranges and summaries read as fluent English.

6. Reply with ONLY this JSON (no transcript text):
     {"output_path": "<OUTPUT>", "failed_windows": <true if '<WORKDIR>/failed_windows.txt' exists, else false>, "warnings": [<any WARNING: line text from step 1>]}
```

## Important rules

- **Never read transcript text into the main (orchestrator) context** — only
  episode subagents touch transcript, SRT bodies, or JSON contents.
- **One subagent per episode; windows run inline** — do not nest subagents.
- **Consistent behavior for 1 or N episodes** — only concurrency varies.
- **Helper owns naming and cleanup** — get paths from `make-workdir`, never build
  output names by hand, and never `rm -rf` a work dir.
- **Quote every dynamic argument** (`shlex.quote` semantics).
- **Summaries are English, not translations** — write fresh English describing
  what's said, not sentence-by-sentence translation.
- **Window-local numbering** — the JSON `start`/`end` always refer to numbers
  `1..M` within the current window. The helper remaps to global indices.
- **Contiguous chunks** — first chunk starts at 1, last ends at M, no gaps or
  overlaps. The helper repairs minor gaps but rejects overlaps.
- **3–15 sentences per chunk** — short enough to summarize tightly, long enough to
  be a meaningful preview.
- **Never delete the Scribe JSON.**
